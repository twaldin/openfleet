# Web Search

Use this skill when the answer depends on current information that may not be in the repo.

1. Define the exact question first so the search can be narrow and testable.
2. Include the current year or date range when freshness matters.
3. Prefer primary sources: official docs, vendor posts, changelogs, standards bodies, and government data.
4. Use secondary sources only to discover leads or compare interpretations.
5. Open multiple sources before concluding, especially when pricing, policy, or breaking changes are involved.
6. Capture publication date and source name for every important claim.
7. Quote exact numbers, limits, flags, or version names instead of paraphrasing them loosely.
8. Distinguish verified facts from your own inference.
9. Flag stale, contradictory, or AI-generated pages instead of treating them as authoritative.

Good search workflow:

- Start broad to find the canonical source.
- Narrow to the exact product, version, region, or API.
- Cross-check at least one independent source if the claim is consequential.

When reporting back:

- Lead with the answer.
- Cite the sources used.
- Mention any uncertainty or date sensitivity.
- Call out what would need re-checking later.

If no trustworthy source exists, say that directly instead of pretending the answer is settled.
